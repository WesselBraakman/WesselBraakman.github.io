My role

API testing standardisation (Postman/Bruno)

Tooling

Context

Teams had inconsistent API checks and ad-hoc collections that were hard to run in CI.

My role

Built a repeatable approach and coached teams on a lightweight standard.

Approach

  • Common structure for requests, environments, and assertions
  • CLI execution in pipelines (reproducible runs)
  • Focus on “fast checks” that catch real integration breaks

Impact

  • Consistent API testing across teams
  • Improved onboarding (less tribal knowledge)
  • Better confidence in contract-level behavior
